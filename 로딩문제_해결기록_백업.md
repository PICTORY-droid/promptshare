# PromptLab 뒤로가기 로딩 문제 해결 기록
날짜: 2025년

---

## 문제 증상
- 프롬프트 상세 페이지에서 뒤로가기 시 메인 페이지가 `loading prompts_` 상태에서 멈춤
- 로컬에서는 정상 작동, 프로덕션(Vercel)에서만 재현
- fetchPrompts 실행 로그는 찍히나 setLoading(false)가 적용 안 됨

---

## 근본 원인 (3가지 복합)

### 원인 1: useState + useEffect 구조의 race condition
- `useState(true)`로 loading 초기값 설정
- 뒤로가기 시 Next.js가 컴포넌트 재마운트
- 비동기 fetch 완료 전 컴포넌트 교체 → `setLoading(false)`가 unmounted 컴포넌트에 적용 → React가 무시
- 새 인스턴스는 `loading = true`로 시작, 아무도 false로 바꾸지 않음

### 원인 2: router.replace()가 Next.js Router Cache 무효화
- URL 파라미터 업데이트에 `router.replace()` 사용
- Next.js 내부 라우팅을 트리거 → Router Cache 무효화
- 뒤로가기 시 컴포넌트 강제 재마운트

### 원인 3: SWR 기본 in-memory 캐시
- SWR은 기본적으로 메모리에 캐시 저장
- 컴포넌트 재마운트 시 캐시 소멸 → `isLoading: true` 재발생

---

## 해결책 (3단계)

### Step 1: useEffect → SWR로 교체 (app/page.tsx)
```tsx
// 기존 (문제)
const [prompts, setPrompts] = useState<Prompt[]>([])
const [loading, setLoading] = useState(true)
const fetchPrompts = async () => {
  setLoading(true)
  // ...fetch...
  setLoading(false)
}
useEffect(() => { fetchPrompts() }, [])

// 변경 (해결)
import useSWR from 'swr'
const { data: swrPrompts, isLoading: loading } = useSWR(
  'prompts',
  async () => {
    const { data, error } = await supabase
      .from('prompts').select('*')
      .order('created_at', { ascending: false }).limit(2000)
    if (error) throw error
    return data ?? []
  },
  { revalidateOnFocus: false, keepPreviousData: true }
)
const prompts = swrPrompts ?? []
```

### Step 2: router.replace() → window.history.replaceState() (app/page.tsx)
```tsx
// 기존 (문제) - Next.js Router 트리거
router.replace(queryStr ? `/?${queryStr}` : '/', { scroll: false })

// 변경 (해결) - Next.js Router 건드리지 않고 URL만 변경
window.history.replaceState(null, '', queryStr ? `/?${queryStr}` : '/')
```

### Step 3: SWR localStorage Provider 적용 (핵심 해결책)
SWR 공식 문서 권장 방식 - 캐시를 localStorage에 영속 저장

**신규 파일: app/components/SWRProvider.tsx**
```tsx
'use client'
import { SWRConfig } from 'swr'

function localStorageProvider() {
  if (typeof window === 'undefined') return new Map()
  const map = new Map<string, any>(
    JSON.parse(localStorage.getItem('pl-swr-cache') || '[]')
  )
  window.addEventListener('beforeunload', () => {
    const cache = JSON.stringify(Array.from(map.entries()))
    localStorage.setItem('pl-swr-cache', cache)
  })
  return map
}

export default function SWRProvider({ children }: { children: React.ReactNode }) {
  return (
    <SWRConfig value={{ provider: localStorageProvider }}>
      {children}
    </SWRConfig>
  )
}
```

**app/layout.tsx에 Provider 적용**
```tsx
import SWRProvider from "./components/SWRProvider"

// body 안에서 children을 SWRProvider로 감싸기
<SWRProvider>
  <Navbar />
  {children}
  ...
</SWRProvider>
```

  // 로딩 UI
) : (
  // 카드 목록

## 빅뱅 페이지 SWR 캐시 공유 (app/bigbang/page.tsx)
```tsx
// 기존: 자체 Supabase fetch (느림, race condition)
useEffect(() => {
  supabase.from('prompts').select(...).then(...)
}, [])

// 변경: 메인 페이지와 동일한 SWR 캐시 공유 (즉시)
import useSWR from 'swr'
const { data: swrData } = useSWR('prompts')
useEffect(() => {
  if (swrData && swrData.length > 0) setPrompts(swrData)
}, [swrData])
```

## 커밋 기록
- `SWR로 데이터 페칭 교체 - 뒤로가기 로딩 문제 근본 해결`

- `뒤로가기 로딩 조건 수정 - 캐시 데이터 있으면 즉시 표시`
- `SWR localStorage provider 적용 - 뒤로가기 캐시 영속화`
- `빅뱅 - 메인 SWR 캐시 공유로 즉시 데이터 사용`
