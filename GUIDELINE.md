## 글 작성 방식

- 한 번 명령 시 20개 작성
- 2단계 방식: 제목 20개 먼저 생성 → 중복 검토 → 통과한 제목으로 내용 생성
- Supabase REST API로 직접 INSERT
- 작성 전 반드시 기존 제목 전체 조회 후 중복 체크
- author_name 항상 PromptLab 고정

## 핵심 원칙 — 항상 최신 데이터 유지
날짜는 사이트에서 숨겼으므로 글 내용 자체가 항상 최신이어야 한다.
이를 위해 모든 수치·법령·정책은 반드시 변수로 처리한다.

나쁜 예 (절대 금지):
- "2024년 최저임금은 9,860원입니다"
- "2025년 부가세율 10% 적용"
- "올해 청약가점 84점 만점"
- "현재 금리는 3.5%"

좋은 예 (반드시 이렇게):
- "현재 적용 최저임금 {{최저임금}}원 기준으로"
- "현행 부가세율 {{세율}}% 기준으로"
- "현재 청약가점제 {{만점}} 기준으로"
- "현행 기준금리 {{금리}}% 기준으로"

규칙:
1. 특정 연도 수치 하드코딩 절대 금지
2. 법령·세율·정책·금리·임금 등 변동 가능한 모든 수치는 변수 처리
3. 공식 출처 링크 제공 (사용자가 직접 최신값 확인 가능하도록)
4. 기존 글 수정 시 고정 수치 → 변수 교체 필수
5. 프롬프트 실행 시점에 사용자가 최신값을 직접 입력하는 구조

## Few-shot 예시 (이 스타일로 작성)
제목: 밀레니얼 직장인이 첫 집 살 때 놓치는 7가지 계약서 함정
설명: 부동산 계약서 서명 전 반드시 확인해야 할 실전 체크리스트. 놓치면 수천만원 손해입니다.
카테고리: Real Estate
작성자명: PromptLab

내용:
프롬프트 1: 계약서 함정 사전 점검
당신은 10년 경력 공인중개사입니다. {{매물유형}}, {{계약금액}}, {{지역}}, {{입주시기}}를 바탕으로 계약서에서 반드시 확인해야 할 독소조항을 찾아주십시오. 현행 주택임대차보호법 기준(https://www.law.go.kr)으로 등기부등본 확인 방법, 선순위 임차인 파악법, 특약사항 필수 문구를 단계별로 안내하십시오.

프롬프트 2: 자금 조달 계획 검토
당신은 부동산 금융 전문가입니다. {{대출한도}}, {{자기자본}}, {{월소득}}을 바탕으로 무리하지 않는 자금 조달 계획을 수립하십시오. 현재 시점 기준금리 {{금리}}%를 적용하여 월 상환액과 위험 시나리오를 포함하십시오.

프롬프트 3: 입지 가치 분석
당신은 부동산 입지 분석 전문가입니다. {{지역}}, {{개발호재}}, {{교통편의성}}을 바탕으로 향후 가치 상승 가능성을 분석하십시오.

프롬프트 4: 협상 전략 수립
당신은 부동산 협상 전문가입니다. {{매도자상황}}, {{매물특성}}, {{시장상황}}을 바탕으로 최적의 가격 협상 전략을 수립하십시오.

## 조회수·좋아요·아쉬워요 기준 (페이지 순서 기반)
날짜가 사이트에 노출되지 않으므로 created_at 순서 기준으로 자연스럽게 설정한다.
AI가 조작한 것처럼 보이지 않도록 들쭉날쭉한 랜덤값을 사용한다.
딱 떨어지는 숫자 절대 금지 (100, 200, 500 같은 숫자 금지)

마지막 페이지 글 (가장 오래된 글):
- views: 500~900 (랜덤, 들쭉날쭉하게)
- likes: views의 4~6% (소수점 버림)
- dislikes: likes의 10~20% (소수점 버림)

중간 페이지 글:
- views: 100~400 (랜덤, 들쭉날쭉하게)
- likes: views의 3~5%
- dislikes: likes의 10~20%

첫 페이지 글 (가장 최신 글):
- views: 10~80 (랜덤, 들쭉날쭉하게)
- likes: views의 2~4%
- dislikes: likes의 10~20%

## 10개마다 체크리스트
- $$ 개수 짝수 확인
- author_id 없는지 확인
- author_name이 PromptLab인지 확인
- 제목에 특정 연도 없는지 확인
- 제목 키워드 3개 이상 겹치지 않는지 확인
- 섹션 4개 이상 확인
- 각 섹션 150자 이상 확인
- 변수 4개 이상 확인
- 고정 수치 없고 변수 처리됐는지 확인
- 조회수가 딱 떨어지는 숫자 아닌지 확인

## 프롬프트 엔지니어링 규칙
1. 역할 부여 → Few-shot 예시 → 핵심 규칙 순서로 지시

3. 제목 20개 먼저 생성 → 중복 검토 → 내용 생성 순서
